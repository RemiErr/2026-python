# T05 測試、日誌與提交文件

## 簡述

建立主線驗收、開發日誌與提交說明，讓專案從可執行提升到可提交。

## 任務目標

完成 `test_chibi.py`、`TEST_LOG.md`、`AI_USAGE.md`，並將主線任務的驗收條件轉成可重跑的測試與記錄。

## 前置依賴

- T02
- T03
- T04

## 輸入

- 主線所有 API 與行為
- 主線所有資料契約

## 產出

- `test_chibi.py`
- `TEST_LOG.md`
- `AI_USAGE.md`

## 必做範圍

- I/O 測試
- 戰鬥邏輯測試
- 報表穩定性測試
- 開發過程日誌
- AI 使用說明

## 非目標

- 不實作 GUI 測試畫面
- 不實作 native 效能測試

## 執行步驟

1. 建立至少 12 個主測試
2. 覆蓋讀檔、EOF、欄位解析、速度排序、最低傷害、三波模擬、排行、勢力統計、戰敗判定、報表不改狀態
3. 將 RED、GREEN、REFACTOR 過程寫入 `TEST_LOG.md`
4. 將 AI 協助範圍寫入 `AI_USAGE.md`

## 驗收標準

- 主線功能都有對應測試
- `TEST_LOG.md` 可回看開發節點
- `AI_USAGE.md` 能說明哪些部分由 AI 協助

## 交接重點

- 若 API 名稱有變更，必須同步更新測試
- 若加入 GUI 或 native，主線測試不可被破壞
- 測試永遠以純 Python 主線作為最低驗收標準
